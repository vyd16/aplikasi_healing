# Build Laravel Project from Scratch

## Goal Description
Create a fresh Laravel application in `c:\xampp\htdocs\hiling` that will serve as the backend for the HealPoint product described in the PRD. The project will include:
- Laravel 11 (latest stable) scaffold
- Database configuration pointing to a MySQL instance (to be created later)
- Initial folder structure for migrations, models, controllers, and Blade views
- A `db/` directory that will later hold a raw SQL dump for easy import to MySQL
- Basic authentication (email/password) using Laravel Sanctum (JWT‑style) as a starting point
- Basic routes and a placeholder home page to verify the installation

## User Review Required
> **[!IMPORTANT]** Confirm that you want the Laravel project generated directly inside `c:\xampp\htdocs\hiling` (overwrites any existing files). If the folder currently contains unrelated files, they will be left untouched, but `composer create-project` will add many new files.
> 
> **[!IMPORTANT]** Approve the Laravel version (default will be the latest stable, currently Laravel 11).
> 
> **[!IMPORTANT]** Approve using Composer for installation (requires Composer to be available on the system). If Composer is not installed, we will need to install it first.

## Verification Plan
### Automated Tests
- Run `composer install` and `php artisan serve` to ensure the development server starts without errors.
- Execute `curl http://127.0.0.1:8000/` and verify a 200 response containing the welcome page.
- Run `php artisan migrate` (with a temporary SQLite DB) to confirm migrations work.

### Manual Verification
- Open a browser to `http://localhost:8000` and confirm the Laravel welcome page is displayed.
- Check that the `.env.example` file exists and contains DB placeholders.
- Verify the `db/` folder is present.
